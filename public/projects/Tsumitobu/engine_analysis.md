# 独自のゲームエンジン（C++ / OpenGLベース）

プロジェクトのディレクトリ構造とソースコードを確認した結果、当プロジェクトはUnityやUnreal Engineのような市販の汎用ゲームエンジンではなく、**C++で独自に開発されたゲームエンジン（カスタムフレームワーク）**を使用しています。

## 主な特徴

### 1. グラフィックスAPI (OpenGL)
コアライブラリとして `GLLibrary` が組み込まれており、描画には **OpenGL** が使用されています。

### 2. 内製ライブラリによる機能実装
3Dゲーム開発に必要なコンポーネントが、クラスとして一通り実装されています（`GLLibrary.h` より）。
* **モデル描画**: 独自形式(A3M)やFBX、Xファイルフォーマットの読み込み (`CModelA3M.h`, `CModelFBX.h`, `CModelX.h`)
* **レンダリングの基本要素**: カメラ(`CCamera.h`)、ライト(`CLight.h`)、シャドウ(`CShadow.h`)
* **各サブシステム**: 当たり判定(`CCollision.h`)、サウンド(`CSound.h`)、入力制御(`CInput.h`)、ネットワーク(`CNetWork.h`)
* **パフォーマンス・最適化**: フレームレート制御(`CFPS.h`)、非同期リソース読み込みスレッド(`CLoadThread.h`)

### 3. 開発環境
* `tumitobu.sln` や `GameProject.vcxproj` が存在することから、**Windows向け**に **Visual Studio (C++)** を用いて開発されています。

## 結論
基礎や低レイヤーから構築された、**フルスクラッチ（手作り）志向のC++ゲーム開発環境**です。専門学校でのチーム制作や、独自のエンジン開発を目的とした独自の学習プロジェクトなどでよく見られる構成となっています。
